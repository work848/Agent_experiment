# Requirements Snapshot

- [pending] R001: 面向零基础学习者的日语学习应用 (P1)
  Description: 构建一款面向日语完全零基础用户的综合学习App，核心目标是帮助外国人在日本求职，覆盖从入门到职业场景训练的完整学习路径。
  Acceptance Criteria:
    1. 应用目标用户被定义为日语完全零基础学习者。
    2. 应用学习目标明确包含“帮助外国人在日本找工作”的实用导向。
    3. 产品内容覆盖从基础学习到职业专项训练的连续学习流程。
    4. 应用支持用户从零开始建立基础后逐步进入求职相关训练。
  Steps: R001-S01, R001-S02

- [pending] R002: 综合课程内容体系 (P1)
  Description: 应用应提供完整的日语学习内容体系，至少包括五十音、词汇、语法、听力和会话模块，形成综合类学习产品。
  Acceptance Criteria:
    1. 课程模块中包含五十音学习内容。
    2. 课程模块中包含词汇学习内容。
    3. 课程模块中包含语法学习内容。
    4. 课程模块中包含听力训练内容。
    5. 课程模块中包含会话训练内容。
  Steps: R002-S01, R002-S02

- [pending] R003: 固定基础学习路线 (P1)
  Description: 应用应为所有用户提供统一的基础学习路线，按照五十音→基础词汇→N5语法的顺序推进，确保用户在进入职业专项训练前具备必要基础。
  Acceptance Criteria:
    1. 所有新用户默认进入统一的基础学习路线。
    2. 基础学习路线顺序固定为“五十音→基础词汇→N5语法”。
    3. 用户在完成基础路线前不能进入职业专项训练内容。
    4. 系统能够记录用户在基础路线中的完成进度。
  Steps: R003-S01, R003-S02

- [pending] R004: 基于学习进度的AI练习生成 (P1)
  Description: 应用应提供AI功能，根据用户当前学习进度动态生成语法和词汇练习内容，确保练习与用户当前能力水平匹配。
  Acceptance Criteria:
    1. AI能够读取用户当前学习进度数据。
    2. AI可生成语法类练习。
    3. AI可生成词汇类练习。
    4. 生成的练习内容与用户当前已学习范围一致，不超出当前阶段过多。
  Steps: R004-S01

- [pending] R005: 基于正确率的AI练习自适应 (P1)
  Description: AI应根据用户练习正确率与学习进度调整练习类型、难度和强化重点，以满足用户对个性化练习方式的需求。
  Acceptance Criteria:
    1. 系统能够记录用户各练习题目的正确率。
    2. AI会根据正确率变化调整后续练习内容。
    3. 当某类知识点正确率较低时，系统会增加该类练习频次。
    4. 当用户学习进度提升时，AI会同步提升练习难度或复杂度。
  Steps: R005-S01, R005-S02

- [pending] R006: 基于艾宾浩斯遗忘曲线的复习安排 (P1)
  Description: 应用应使用艾宾浩斯遗忘曲线为用户自动安排复习时间，提升记忆保持效果。
  Acceptance Criteria:
    1. 系统可根据用户学习记录生成复习计划。
    2. 复习提醒或复习任务安排遵循艾宾浩斯遗忘曲线逻辑。
    3. 用户完成学习后，系统会自动创建后续复习节点。
    4. 用户可在应用内查看待复习内容及对应时间安排。
  Steps: R006-S01, R006-S02

- [pending] R007: 职场导向专项训练 (P1)
  Description: 应用应在用户完成基础能力建设后，提供与日本求职相关的专项训练内容，包括面试、简历和职业相关词汇学习。
  Acceptance Criteria:
    1. 专项训练内容包含面试场景训练。
    2. 专项训练内容包含简历相关内容训练。
    3. 专项训练内容包含职业相关词汇训练。
    4. 用户只有在完成基础路线后才能解锁职场专项训练。
  Steps: R007-S01, R007-S02

- [pending] R008: 就业倾向采集与路线推荐 (P2)
  Description: 应用应要求用户先表达就业倾向，并基于该倾向在基础路线完成后推荐对应的职业学习路线。
  Acceptance Criteria:
    1. 用户首次使用或进入职业规划阶段时需要填写就业倾向信息。
    2. 系统能够保存用户的就业倾向。
    3. 基础路线完成后，系统会根据就业倾向推荐对应职业路线。
    4. 不同就业倾向可触发不同的后续专项学习内容。
  Steps: R008-S01, R008-S02

- [pending] R009: 职业路线在基础后分化 (P2)
  Description: 应用应在固定基础路线之后，根据用户不同职业方向提供差异化学习内容和训练重点。
  Acceptance Criteria:
    1. 所有用户在基础阶段学习内容一致。
    2. 基础阶段结束后，系统可根据职业方向切换后续内容。
    3. 不同职业路线至少在词汇、场景或练习内容上存在差异。
    4. 系统能够明确展示用户当前所属职业学习路线。
  Steps: R009-S01, R009-S02

- [pending] R010: 错误知识点的情景会话强化 (P2)
  Description: 当用户在某一知识点上持续出错时，系统应优先采用情景会话方式帮助用户理解和巩固该知识点。
  Acceptance Criteria:
    1. 系统能够识别用户在同一知识点上的持续错误行为。
    2. 当某知识点达到持续错误阈值时，系统会触发情景会话式练习。
    3. 情景会话内容与该错误知识点直接相关。
    4. 用户可在对应知识点下收到针对性的情景理解练习。
  Steps: R010-S01, R010-S02

- [pending] R011: 以翻译题为主的练习模式 (P2)
  Description: 应用的练习设计应以翻译题为主，选择题为辅助，并在一定学习阶段后加入情景对话模拟。
  Acceptance Criteria:
    1. 练习系统中提供翻译题型，并作为主要题型出现。
    2. 练习系统中提供选择题型，并作为辅助题型使用。
    3. 系统可根据学习阶段控制题型比例。
    4. 用户达到设定阶段后，系统会开放情景对话模拟练习。
  Steps: R011-S01, R011-S02

- [pending] R012: 阶段性情景对话模拟 (P3)
  Description: 应用应在用户达到一定学习阶段后提供情景对话模拟，帮助用户将基础知识应用到真实交流和求职场景中。
  Acceptance Criteria:
    1. 系统为情景对话模拟设置明确的解锁阶段条件。
    2. 达到条件后，用户可进入情景对话模拟功能。
    3. 情景对话模拟内容可结合日常交流或求职场景。
    4. 用户能够在模拟中进行多轮对话练习。
  Steps: R012-S01, R012-S02

- [pending] R013: 个性化复习与加强训练 (P2)
  Description: AI应支持为用户定制复习和强化训练方案，重点补足用户薄弱点，并满足用户对特定练习方式的个性化需求。
  Acceptance Criteria:
    1. 系统能够识别用户的薄弱知识点。
    2. AI可针对薄弱点生成专属复习任务。
    3. AI可根据用户表现调整强化训练内容。
    4. 用户获得的复习与训练计划与个人学习记录相关联。
  Steps: R013-S01, R013-S02
